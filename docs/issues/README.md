# Issues (file-based tracker)

This directory holds issue specs while the repo has no GitHub remote. Each
file follows the agent-brief format from the `triage` skill so it can be
copy-pasted straight into a GitHub Issue body the moment we add a remote.

## Conventions

- One issue per file, named `NNN-short-slug.md`, monotonically numbered.
- Front matter line `> *This was generated by AI during triage.*` matches
  the disclaimer the triage skill posts on real GitHub comments.
- Labels are tracked inline in the body (`**Category:**`, `**State:**`)
  since there's no Issues UI to attach them to yet.
- When we push to GitHub, port each file to an Issue and apply the labels
  from its front matter.

## State legend

- `needs-triage` — never evaluated
- `needs-info` — waiting on more info from reporter
- `ready-for-agent` — fully specified, an AFK agent can pick it up
- `ready-for-human` — needs human judgement
- `wontfix` — won't be actioned
